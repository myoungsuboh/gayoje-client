# 📋 Implementation Plan & Progress Tracking Guide

> 新しい機能開発やバグ修正を始める前に実装計画を立て、進捗状況を追跡するための標準。作業の全体像をつかみ、進捗をチームと共有する必要があるときに読みます。

## 1. 基本原則
- コーディングの前に実装計画を先に立てて道に迷うことを防ぐ。
- 大きな作業はUI・状態管理・API連携など小さな単位に分割する。
- 進捗状態を標準表記(`[ ]`/`[-]`/`[x]`/`[!]`)で可視化して共有する。
- 計画段階で詰まる箇所と課題を事前に発見する。

## 2. ルール

### 2-1. なぜ実装計画を書くべきなのか?(ジュニア開発者向けのヒント)
1. **道に迷うのを防止:** コーディングから始めると詳細な実装にのめり込み、目的を忘れやすくなります。計画を先に立てるとナビゲーションの役割を果たしてくれます。
2. **作業の分割 (Divide & Conquer):** 途方もなく見える大きな作業も、UI、状態管理、API連携など小さな単位に分割すると、取り組める大きさになります。
3. **進捗度の可視化:** 自分が一日中何をしたのか、現在どこで止まっているのかを、チームメンバー(特に先輩)に簡単に共有できます。
4. **課題の事前遮断:** 計画を立てる段階で「あれ?このデータはどこから取得するんだ?」といった詰まる箇所を事前に発見できます。

### 2-2. 進捗状態(Status)表記標準
* `[ ]` ⚪ **TODO**: まだ着手していない
* `[-]` 🟡 **IN PROGRESS**: 現在作業中
* `[x]` 🟢 **DONE**: 作業完了および自己テスト通過
* `[!]` 🔴 **BLOCKED**: 外部要因により作業が止まっている

### 2-3. Implementation Plan 標準テンプレート

#### 1. 📝 作業概要 (Task Overview)
* **作業目標:** (例: 共通日時ピッカー `<AppDateTimePicker>` コンポーネントの実装)
* **関連課題/チケット:** `#123`
* **想定スケジュール:** `2026.03.27 ~ 2026.03.29`

#### 2. 🔍 事前確認事項 (Prerequisites)
- [ ] Figmaデザイン案の確認
- [ ] バックエンドAPI仕様書(Swagger)の更新有無の確認
- [ ] 既存の類似コンポーネントの有無の把握(重複開発の防止)

#### 3. 🏃‍♂️ 段階別実装計画 (Step-by-step Plan)

- **Phase 1: UIおよびコンポーネントの骨格づくり**
  - [x] 🟢 ファイル生成
  - [-] 🟡 基本UIマークアップ

- **Phase 2: 状態管理およびロジック連携**
  - [ ] ⚪ `v-model` バインディング処理
  - [ ] ⚪ Validationロジックの追加

- **Phase 3: API連携**
  - [ ] ⚪ Mockデータテスト
  - [ ] ⚪ 実際のAPI連携およびエラーハンドリング

#### 4. ✅ 最終検証 (QA & Testing)
- [ ] ⚪ デスクトップおよびモバイル環境のUI確認
- [ ] ⚪ コンソールエラー/警告がないことの確認
- [ ] ⚪ `npm run build` エラーがないことの確認

## 3. よくある間違い
- ❌ 計画なしにすぐコーディング → 目的を見失って迷走する。まず段階計画を立てる。
- ❌ 巨大な段階を一塊にする → 進捗・課題が見えない。UI/状態/API単位に分割する。
- ❌ 段階別の完了(検証)基準の欠落 → 「終わった」という判断が曖昧。各段階に検証項目を置く。
- ❌ BLOCKEDを放置・未共有 → 先輩が詰まりを知らない。外部要因を明示して共有する。
- ❌ 計画を立てるだけで状態を更新しない → ドキュメントが現実と乖離する。進捗ごとに表記を更新する。
- ❌ ロールバック・リスクを考慮しない → 失敗時の対応がない。

## 4. チェックリスト
- [ ] コーディングの前に作業概要(目標・課題・スケジュール)を書いた
- [ ] 事前確認事項(デザイン案、API仕様、類似コンポーネント)を点検した
- [ ] 作業をPhase単位(UI/状態/API)に分割した
- [ ] 各項目に標準状態表記(`[ ]`/`[-]`/`[x]`/`[!]`)を適用する
- [ ] BLOCKED項目は外部要因を明示して共有する
- [ ] 最終検証(デスクトップ/モバイルUI、コンソールエラー、`npm run build`)を通過した
