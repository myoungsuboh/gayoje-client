# 🤖 AIエージェントルールファイル (AGENTS.md)

> AIコーディングエージェントにリポジトリのルートのルールファイルを読ませ、プロジェクトの規約どおりに作業させる。新しいプロジェクトを始めるときやバイブコーディングのコンテキストを整備するとき、エージェントがコマンド・規約を知らず迷うときに読む。

AIコーディングエージェント（Cursor, Claude Code, Windsurf, Codex, Copilot）は、リポジトリのルートの **AGENTS.md** を
ネイティブに読み、プロジェクト別の作業指針とする。2026年に6万以上の公開リポジトリが採用した事実上の標準である。

## 1. 核心原則

- リポジトリのルートに `AGENTS.md`（または `CLAUDE.md`/`.cursorrules`）を置き、プロジェクト概要・技術スタック・セットアップコマンド・コードスタイル・テスト方法・セキュリティ注意事項のセクションを設ける。
- エージェントがコードから自力で分からないことだけを書く — ビルド/実行/テストコマンド、規約の「なぜ」、やってはいけないこと（反例）に集中する。
- 抽象的な説明三段落より実際のコードスニペット一つが優れる — 望むパターンを例示コードで示す。
- 破壊的・非可逆な操作（DBマイグレーション、デプロイ、シークレット変更、大量削除）は「人の確認なしに実行禁止」と明示する。
- ルールファイルは人が直接キュレーションする — 自動生成で肥大化するとトークンだけ食って精度が落ちる。短く正確に保つ。
- 複数のエージェントを使うなら `AGENTS.md` を単一の真実の源として置き、`CLAUDE.md`/`.cursorrules` はシンボリックリンクや参照で統一する。

## 2. ルール

### 2-1. なぜ必要か

エージェントはコードを読んで*構造*は把握するが、**ビルド/実行/テストコマンド**、**規約の意図**、
**してはいけないこと**は推論できない。これを毎セッション繰り返し説明しないよう、ファイルで固定する。

### 2-2. 推奨構造（コミュニティ収束型）

```markdown
# <프로젝트명>

## Project Overview
한 문단으로 무엇을 만드는 서비스인지.

## Tech Stack
- Frontend: Vue 3 + Vite + Vuetify
- Backend: Spring Boot 3 (Java 17)
- DB: PostgreSQL + MyBatis

## Setup / Commands
- 설치: `pnpm i`
- 개발: `pnpm dev`
- 테스트: `pnpm test` (반드시 통과 후 커밋)
- 빌드: `pnpm build`

## Code Style
- 컴포넌트 PascalCase, 변수/함수 camelCase
- 커밋: `type(scope): 요약` (Conventional Commits)
- (반례) any 타입 금지 — unknown + 좁히기 사용

## Testing
- 변경한 코드 경로의 테스트를 추가/갱신하고 전체 그린 확인 후 커밋

## Security
- 시크릿은 .env (커밋 금지). 결제/인증 로직은 사람 검토 필수
- DB 마이그레이션·배포·대량 삭제는 사람 확인 없이 실행 금지
```

### 2-3. 作成原則

1. **エージェントが知らないことだけを書く。** コードで自明なものは外し、コマンド・意図・禁止事項に集中。
2. **反例（counterexample）が最も価値がある。** 「any 禁止」「このフォルダは自動生成なので修正禁止」のような直観に反するルール。
3. **スニペット > 説明。** 望むパターンをコードで示すと、冗長な説明より正確に従う。
4. **短く保つ。** 膨大な自動生成ファイルはトークンだけ食ってかえって精度を落とす — 人がキュレーションする。

### 2-4. 複数エージェントの一元化

```bash
# ✅ 권장 — AGENTS.md 를 단일 진실원으로 두고 나머지는 심볼릭 링크
ln -s AGENTS.md CLAUDE.md
ln -s AGENTS.md .cursorrules
```

## 3. よくある間違い

- ファイル一つに数百行の一般論（エージェントが既に知っている React 構文など）を詰め込む → トークンだけ食って精度が落ちる。
- 頻繁に変わる詳細な実装を書く → すぐに stale になる。安定した規約・コマンド中心に書く。
- 破壊的操作の禁止ルールを書き忘れる → エージェントが DB マイグレーション・デプロイを勝手に実行する。
- エージェントごとのルールファイルを別々に管理する → 内容が食い違う。単一の真実の源 + リンクで統一する。
