# 🗂️ 空状態 · エラー · ローディング状態 UI

> 非同期画面のローディング・成功・エラー・空状態を一貫して処理する。非同期コンポーネントを作るときや状態ごとの表示を決めるときに読む。

## 1. 基本原則
- すべての非同期 UI 領域は、ローディング・成功・エラー・空状態の4つを明示的に処理する。
- ローディング状態は実際のコンテンツレイアウトと同一の Skeleton UI を使い、レイアウトのジャンプを防ぐ。
- エラー状態は再試行ボタンを含み、全ページエラーと部分(インライン)エラーを区別する。
- 空状態は単なる 'No data' ではなく、理由・行動喚起・イラストで構成する。
- 楽観的更新(Optimistic UI)を適用する際は、ロールバックシナリオとエラー復旧を必ず実装する。

## 2. ルール

### 2-1. 状態マトリクス
すべての非同期コンポーネントは4つの状態を処理しなければならない。

| 状態 | 表示 | 主要素 |
|------|------|-----------|
| Loading | Skeleton / Spinner | コンテンツの形を維持 |
| Success | 実際のコンテンツ | — |
| Error | エラーメッセージ | 再試行ボタン |
| Empty | 空状態 UI | 理由 + 行動喚起 |

### 2-2. Skeleton UI パターン
```html
<!-- ❌ 금지 — 단순 스피너만 두면 로딩 후 레이아웃이 점프함 -->
<div class="spinner">Loading...</div>

<!-- ✅ 권장 — 실제 카드와 동일한 구조 유지 -->
<div class="card skeleton" aria-busy="true" aria-label="콘텐츠 불러오는 중">
  <div class="skeleton-line" style="width: 60%; height: 20px;"></div>
  <div class="skeleton-line" style="width: 90%; height: 16px;"></div>
  <div class="skeleton-block" style="height: 120px;"></div>
</div>
```

### 2-3. エラー状態の階層
全ページエラーと部分(インライン)エラーを区別する。

インラインエラー(部分的失敗):
```
⚠️ [항목 이름]을 불러오지 못했습니다.
   [다시 시도] 버튼
```

全ページエラー:
```
😞 페이지를 불러올 수 없어요.
   인터넷 연결을 확인하거나 잠시 후 다시 시도해 주세요.
   [다시 시도] [홈으로]
```

### 2-4. 空状態 UI の構成
```
[일러스트 또는 아이콘]

[제목] 아직 {항목}이 없어요

[설명] {항목}을 추가하면 여기에 표시됩니다.

[CTA 버튼] + 첫 {항목} 만들기
```

空状態のタイプ別処理:
- **初回利用**: オンボーディング案内を含める
- **検索結果なし**: 検索語の修正を提案
- **権限なし**: 権限のリクエスト方法を案内
- **フィルタ結果なし**: フィルタのリセットボタンを提供

## 3. よくあるミス
- ローディングにスピナーのみを使用 → コンテンツ到着時にレイアウトがジャンプする。
- エラーをすべて全ページとして処理 → 部分的失敗でも画面全体が塞がれる。
- 空状態に 'No data' のみを表示 → ユーザーが次の行動を分からない。
- 楽観的更新後にロールバック未実装 → 失敗しても成功したように見える。
