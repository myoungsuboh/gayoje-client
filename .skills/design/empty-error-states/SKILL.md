# 🗂️ 빈 상태 · 오류 · 로딩 상태 UI

> 비동기 화면의 로딩·성공·오류·빈상태를 일관되게 처리한다. 비동기 컴포넌트를 만들거나 상태별 표시를 정할 때 읽는다.

## 1. 핵심 원칙
- 모든 비동기 UI 영역은 로딩·성공·오류·빈상태 네 가지를 명시적으로 처리한다.
- 로딩 상태는 실제 콘텐츠 레이아웃과 동일한 Skeleton UI를 사용해 레이아웃 점프를 방지한다.
- 오류 상태는 재시도 버튼을 포함하고, 전체 페이지 오류와 부분(인라인) 오류를 구분한다.
- 빈 상태는 단순 'No data'가 아닌 이유·행동 유도·일러스트로 구성한다.
- 낙관적 업데이트(Optimistic UI) 적용 시 롤백 시나리오와 오류 복구를 반드시 구현한다.

## 2. 규칙

### 2-1. 상태 매트릭스
모든 비동기 컴포넌트는 4가지 상태를 처리해야 한다.

| 상태 | 표시 | 핵심 요소 |
|------|------|-----------|
| Loading | Skeleton / Spinner | 콘텐츠 형태 유지 |
| Success | 실제 콘텐츠 | — |
| Error | 오류 메시지 | 재시도 버튼 |
| Empty | 빈 상태 UI | 이유 + 행동 유도 |

### 2-2. Skeleton UI 패턴
```html
<!-- ❌ 금지 — 단순 스피너만 두면 로딩 후 레이아웃이 점프함 -->
<div class="spinner">Loading...</div>

<!-- ✅ 권장 — 실제 카드와 동일한 구조 유지 -->
<div class="card skeleton" aria-busy="true" aria-label="콘텐츠 불러오는 중">
  <div class="skeleton-line" style="width: 60%; height: 20px;"></div>
  <div class="skeleton-line" style="width: 90%; height: 16px;"></div>
  <div class="skeleton-block" style="height: 120px;"></div>
</div>
```

### 2-3. 오류 상태 계층
전체 페이지 오류와 부분(인라인) 오류를 구분한다.

인라인 오류 (부분 실패):
```
⚠️ [항목 이름]을 불러오지 못했습니다.
   [다시 시도] 버튼
```

전체 페이지 오류:
```
😞 페이지를 불러올 수 없어요.
   인터넷 연결을 확인하거나 잠시 후 다시 시도해 주세요.
   [다시 시도] [홈으로]
```

### 2-4. 빈 상태 UI 구성
```
[일러스트 또는 아이콘]

[제목] 아직 {항목}이 없어요

[설명] {항목}을 추가하면 여기에 표시됩니다.

[CTA 버튼] + 첫 {항목} 만들기
```

빈 상태 유형별 처리:
- **최초 사용**: 온보딩 안내 포함
- **검색 결과 없음**: 검색어 수정 제안
- **권한 없음**: 권한 요청 방법 안내
- **필터 결과 없음**: 필터 초기화 버튼 제공

## 3. 흔한 실수
- 로딩에 스피너만 사용 → 콘텐츠 도착 시 레이아웃이 점프한다.
- 오류를 전부 전체 페이지로 처리 → 부분 실패까지 화면 전체가 막힌다.
- 빈 상태에 'No data'만 표시 → 사용자가 다음 행동을 모른다.
- 낙관적 업데이트 후 롤백 미구현 → 실패해도 성공한 것처럼 보인다.

## 4. 체크리스트
- [ ] 비동기 영역마다 로딩·성공·오류·빈상태 4가지를 처리했는가
- [ ] 로딩에 콘텐츠와 같은 구조의 Skeleton을 사용했는가
- [ ] 오류에 재시도 버튼이 있고, 인라인/전체 오류를 구분했는가
- [ ] 빈 상태에 이유·행동 유도(CTA)를 넣었는가
- [ ] 낙관적 업데이트 시 롤백·오류 복구를 구현했는가
