# 🗂️ 空状态 · 错误 · 加载状态 UI

> 一致地处理异步页面的加载·成功·错误·空状态。在构建异步组件或确定各状态显示时阅读。

## 1. 核心原则
- 每个异步 UI 区域都显式处理加载·成功·错误·空状态四种情况。
- 加载状态使用与实际内容布局相同的 Skeleton UI，以防止布局跳动。
- 错误状态包含重试按钮，并区分整页错误与局部(内联)错误。
- 空状态不是简单的 'No data'，而是由原因·行动引导·插图构成。
- 应用乐观更新(Optimistic UI)时，务必实现回滚场景与错误恢复。

## 2. 规则

### 2-1. 状态矩阵
每个异步组件都必须处理四种状态。

| 状态 | 显示 | 核心要素 |
|------|------|-----------|
| Loading | Skeleton / Spinner | 保持内容形态 |
| Success | 实际内容 | — |
| Error | 错误消息 | 重试按钮 |
| Empty | 空状态 UI | 原因 + 行动引导 |

### 2-2. Skeleton UI 模式
```html
<!-- ❌ 금지 — 단순 스피너만 두면 로딩 후 레이아웃이 점프함 -->
<div class="spinner">Loading...</div>

<!-- ✅ 권장 — 실제 카드와 동일한 구조 유지 -->
<div class="card skeleton" aria-busy="true" aria-label="콘텐츠 불러오는 중">
  <div class="skeleton-line" style="width: 60%; height: 20px;"></div>
  <div class="skeleton-line" style="width: 90%; height: 16px;"></div>
  <div class="skeleton-block" style="height: 120px;"></div>
</div>
```

### 2-3. 错误状态层级
区分整页错误与局部(内联)错误。

内联错误(局部失败):
```
⚠️ [항목 이름]을 불러오지 못했습니다.
   [다시 시도] 버튼
```

整页错误:
```
😞 페이지를 불러올 수 없어요.
   인터넷 연결을 확인하거나 잠시 후 다시 시도해 주세요.
   [다시 시도] [홈으로]
```

### 2-4. 空状态 UI 构成
```
[일러스트 또는 아이콘]

[제목] 아직 {항목}이 없어요

[설명] {항목}을 추가하면 여기에 표시됩니다.

[CTA 버튼] + 첫 {항목} 만들기
```

按空状态类型处理:
- **首次使用**: 包含引导(onboarding)说明
- **无搜索结果**: 建议修改搜索词
- **无权限**: 说明申请权限的方法
- **无筛选结果**: 提供筛选重置按钮

## 3. 常见错误
- 加载只使用 spinner → 内容到达时布局跳动。
- 将所有错误都按整页处理 → 连局部失败也会让整个屏幕被阻塞。
- 空状态只显示 'No data' → 用户不知道下一步行动。
- 乐观更新后未实现回滚 → 即使失败也看起来像成功。

## 4. 检查清单
- [ ] 是否为每个异步区域处理了加载·成功·错误·空状态四种情况？
- [ ] 加载是否使用了与内容结构相同的 Skeleton？
- [ ] 错误是否有重试按钮，并区分内联/整页错误？
- [ ] 空状态是否加入了原因·行动引导(CTA)？
- [ ] 乐观更新时是否实现了回滚·错误恢复？
