# ♿ ウェブアクセシビリティ — WCAG 2.1 AA

> WCAG 2.1 AA 基準で誰もが使える UI を作るためのスタック中立な基準(SoT)である。新しい画面・コンポーネントを作る、アクセシビリティを点検するときに読む。
>
> Vue/Vuetify 実装(コンポーネントマークアップ、Focus Trap、axe/Playwright、reduced-motion)は `accessibility-a11y` を参照。

## 1. 中核原則

POUR — すべてのアクセシビリティ基準はこの 4 つに還元される。

| 原則 | 意味 | 主な基準 |
|------|------|-----------|
| Perceivable | 知覚可能 | alt text, 色のコントラスト, キャプション |
| Operable | 操作可能 | キーボード, 十分な時間, 発作リスクなし |
| Understandable | 理解可能 | 言語宣言, 一貫したナビゲーション, エラー案内 |
| Robust | 堅牢 | セマンティック HTML, ARIA の正しい使用 |

中核規約(詳細は §2):
- すべてのインタラクティブ要素はキーボード(Tab·Enter·Escape·方向キー)だけで操作できなければならない。
- 画像には意味のある alt を、装飾用画像には `alt=""` を使う。アイコンボタン・リンクにはアクセシブルな名前を提供する。
- 色だけで情報を伝えない(色 + アイコン/テキストを併用)。
- モーダル・ドロワーなどのオーバーレイは focus trap を実装し、Esc で閉じる。

## 2. ルール

### 2-1. セマンティックマークアップ
```html
<!-- ❌ 禁止 — 意味のない div、スクリーンリーダーがボタンだと分からない -->
<div class="btn" onclick="...">저장</div>

<!-- ✅ 推奨 — セマンティックタグで役割を伝える -->
<button type="button">저장</button>
```
- `<div>` の代わりに `<button>`, `<nav>`, `<main>`, `<header>`, `<footer>` のセマンティックタグを使う。
- 見出しの階層構造を維持する (`h1` → `h2` → `h3`、スキップ禁止)。
- `lang="ko"` (または該当言語) を宣言する。

### 2-2. キーボードナビゲーション
```css
/* ❌ 禁止 — フォーカスが見えない */
:focus { outline: none; }

/* ✅ 推奨 — フォーカススタイルを維持 */
:focus-visible { outline: 2px solid var(--color-focus-ring); }
```
```html
<!-- モーダル focus trap の例 -->
<dialog role="dialog" aria-modal="true" aria-labelledby="dialog-title">
  <h2 id="dialog-title">...</h2>
  <!-- フォーカス可能な要素 -->
  <button>닫기</button>
</dialog>
```
- 正の `tabindex` (`tabindex="5"`) はタブ順序を壊すので禁止する。

### 2-3. ARIA 使用原則
```html
<!-- ❌ 禁止 — アイコンのみ、名前なし -->
<button><svg .../></button>

<!-- ✅ 推奨 — アクセシブルな名前を提供 -->
<button aria-label="닫기"><svg aria-hidden="true" .../></button>
```
- HTML セマンティックで解決できるなら ARIA は追加しない。
- `aria-live="polite"` — 動的コンテンツ変更の通知 (ローディング完了、エラーなど)。
- `aria-expanded`, `aria-controls` — アコーディオン・ドロップダウンの状態を伝える。

### 2-4. 色のコントラスト基準

この表が色のコントラスト数値の単一の定義出所(SoT)である。

| テキスト種別 | 最小コントラスト |
|-------------|-----------|
| 通常テキスト | 4.5 : 1 |
| 大型テキスト (18pt 通常 / 14pt Bold) | 3 : 1 |
| 非テキスト UI (アイコン・入力枠) | 3 : 1 |

## 3. よくあるミス
- `outline: none` でフォーカス表示を消す → キーボードユーザーが位置を見失う。
- アイコンボタンに名前を提供しない → スクリーンリーダーが「ボタン」としか読まない。
- 見出し階層のスキップ(h1 → h4) → 文書構造が崩れる。
- 色だけで情報を伝える → 色覚異常・弱視のユーザーが区別できない。

- [ ] 画像 alt / アイコンボタンのアクセシブルな名前を提供したか
