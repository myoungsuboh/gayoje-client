# ✍️ UX 文案 & 微文案

> 以用户为中心统一 UI 中的所有文本。编写新页面的文案，或确定按钮、错误、加载、空状态文本时阅读。

## 1. 核心原则
- 按钮文本使用以动词开头的、面向行动的措辞（例如：'保存'、'删除'、'开始使用'）。
- 错误信息同时提供原因和解决方法，且不暴露技术术语（500 Error、null、undefined）。
- 加载状态用'加载中...'、'保存中...'等明示正在进行，让用户知道有响应。
- 空状态（Empty State）要提供原因 + 行动引导文案，不要只显示'无数据'。
- 面向国际化（i18n）的文本不要硬编码，必须用翻译键管理。

## 2. 规则

### 2-1. 按钮 & 标签
| ❌ 反面示例 | ✅ 正面示例 |
|---------|---------|
| 确定 | 保存 |
| 取消 | 返回 |
| 删除 | 删除账户 |
| Error | 保存失败 |

- **具体性**：明确这是做什么的按钮
- **一致性**：同一行动用同一词（禁止混用"保存"与"保存它"）
- **简洁性**：最多 3~4 个词

### 2-2. 错误信息模式
```
❌ 나쁜 예: "Error 422: Validation failed"
✅ 좋은 예: "이메일 형식이 올바르지 않습니다. 예: example@domain.com"

❌ 나쁜 예: "네트워크 오류"
✅ 좋은 예: "연결에 실패했습니다. 인터넷 연결을 확인하고 다시 시도해 주세요."
```

错误信息的构成：
1. **什么** 出了问题（原因）
2. **如何** 解决（行动）
3. 必要时提供帮助链接

### 2-3. 空状态（Empty State）模板
```
[이유] 아직 {항목}이 없어요.
[유도] {첫 번째 항목}을 만들어 시작해 보세요.
[행동] [+ {항목} 만들기] 버튼
```

### 2-4. 编写占位符
- 使用输入示例（`예: hello@example.com`）—— 格式提示
- 禁止重复标签（`输入姓名` → 标签已是 `姓名`，故只放示例）
- 辅助提示用 `<small>` 或 `helper-text` 显示在标签下方

## 3. 常见错误
- '确定'、'取消'之类模糊的按钮 → 无法得知会发生什么。
- 同一行动混用'保存'和'保存它' → 一致性被破坏。
- 直接暴露技术错误码（500、null）→ 用户不知道怎么解决。
- 空状态只显示'无数据' → 无法引导下一步行动。
- 硬编码文本 → 国际化和文案修改变得困难。

## 4. 检查清单
- [ ] 按钮是否以动词开头且为具体的行动措辞
- [ ] 是否对同一行动一致地使用了同一词
- [ ] 错误信息是否有原因 + 解决方法且无技术术语
- [ ] 空状态是否有原因 + 行动引导文案
- [ ] 是否用翻译键管理了 i18n 目标文本
