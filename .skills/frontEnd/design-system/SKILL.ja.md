# 🎨 デザインシステム (Design System)

> デザイントークンを**Webにどう適用するか**を扱う — スタイルの優先順位、テーマ（ライト/ダーク）切り替え、共通ビジュアルパターンの標準化。新しい画面/コンポーネントのスタイルを決めるとき、テーマ・ダークモードを設計するとき、ハードコードされた値をトークンに整理するときに読む。特定の UI フレームワーク/ライブラリに依存しない汎用標準である。
>
> **トークンの値・命名・階層（primitive→semantic→component）を定義する権威（SoT）は `design-system-tokens` スキルである。** このスキルはそのトークンを Web で消費する規則だけを扱う。

## 1. 核心原則
- **トークンの定義は `design-system-tokens` に従う**: トークンの単一の出所、セマンティック変数、primitive→semantic→component の階層、命名はそのスキル（SoT）の規則である。本スキルは定義済みトークンを画面で参照することだけを扱う — コンポーネントはセマンティックトークンのみで塗り、原始値・原始パレットを直接埋め込まない。
- **テーマはトークンの再定義で切り替える**: ライト/ダークなどのテーマ差はセマンティックトークンの値をテーマごとに変えて表現する。コンポーネントのコードはテーマを知らないようにし、分岐（`if dark`）を撒き散らさない。
- **一貫したスタイルの優先順位**: トークン/システムユーティリティ → 共通変数 → コンポーネントのローカルスタイル の順で適用する。同じ判断を複数の階層でばらばらに下さない。
- **共通ビジュアルパターンは標準化する**: 繰り返し現れるビジュアル効果（半透明な面、モーション/アニメーションなど）は毎回作り直さず、共通のトークン・ユーティリティ・ミックスインとして定義して再利用する。
- **アクセシビリティはデフォルト**: 色のコントラスト・モーション低減（`prefers-reduced-motion`）などのアクセシビリティ制約をトークン・パターンの段階で保証する。

> 任意の値（マジックナンバー）の禁止は、トークンの次元では `design-system-tokens`（トークンになければトークンを増やす）、名前付き定数の次元では `coding-styles` スキルに従う。本スキルの §2-3 はこれをスタイルの優先順位の観点からのみ扱う。

## 2. 規則

### 2-1. テーマ（ライト/ダーク）はトークンの再定義で切り替える
- テーマ差はセマンティックトークン値の差として表現し、テーマスイッチは一箇所（ルート/テーマスコープ）でトークンを再定義して掛ける。
- コンポーネント内に `if (dark) ... else ...` の分岐を撒き散らさない。色は常に同じセマンティックトークンで塗る。（トークン名・値そのものは `design-system-tokens` に従う。）

```text
// ✅ 推奨 — テーマごとにセマンティックトークン値のみ再定義、コンポーネントはそのまま
:root             { --surface-bg: #fff; --text-default: #111; }
[data-theme=dark] { --surface-bg: #111; --text-default: #eee; }

card { background: var(--surface-bg); color: var(--text-default); }  // テーマを知らない

// ❌ 禁止 — コンポーネントがテーマを知って分岐
card { background: isDark ? #111 : #fff; }
```

### 2-2. スタイルの優先順位を守る
- 同じスタイル判断を複数の階層で重複して下さない。優先順位:
  1. **デザインシステムのトークン / 標準ユーティリティ**（間隔・タイポ・整列など — システムが提供するなら優先使用）
  2. **共通変数**（セマンティックトークン/共通スタイル変数を参照）
  3. **コンポーネントのローカルスタイル**（上で表現できない場合にのみ、スコープを狭めて記述）
- ローカルスタイルでトークンを無視して値を埋め直さない。

```text
// ✅ 推奨 — システムユーティリティ/トークン優先、不足分のみローカルスタイル
<div class="stack gap-3">…</div>            // 標準間隔ユーティリティ
.special { box-shadow: var(--shadow-md); }   // トークンを参照したローカルスタイル

// ❌ 禁止 — ローカルスタイルでシステムを迂回してマジックナンバー
.special { margin-top: 13px; box-shadow: 0 2px 7px #0003; }
```

### 2-3. 共通ビジュアルパターンは標準化して再利用する
- 繰り返し現れるビジュアル効果（半透明/ブラーの面、モーションなど）はトークン・ユーティリティ・ミックスインとして定義して再利用する。
- 同じ効果を画面ごとに手で作り直さない — 値が微妙にずれて一貫性が崩れる。

```text
// ✅ 推奨 — パターンをトークン/ユーティリティで標準化
.surface-glass {           // 半透明な面パターンを一度定義して再利用
  background: var(--surface-overlay);
  backdrop-filter: blur(var(--blur-md));
  border: 1px solid var(--border-subtle);
}

// ❌ 禁止 — 画面ごとに直接数値を埋めてばらばら
.panel { background: rgba(255,255,255,.15); backdrop-filter: blur(11px); }
```

### 2-4. アクセシビリティをパターンに組み込む
- テキスト/背景の色コントラストが基準（例: WCAG AA）を満たすようにセマンティックトークンのペアを定義する。
- モーションパターンは `prefers-reduced-motion` を尊重し、ユーザーがモーション低減を望めば無効化する。

```text
// ✅ 推奨 — モーション低減の選好を尊重
@media (prefers-reduced-motion: reduce) {
  * { animation: none; transition: none; }
}
```

## 3. よくある間違い
- **コンポーネントがテーマで分岐** → 画面ごとに `if dark` が散らばり、新しいテーマを追加するたびにすべてのコンポーネントを直す必要がある。トークンの再定義に移す。
- **優先順位の無視** → システムユーティリティ/トークンがあるのにローカルスタイルで値を埋め直し、衝突・重複が生じる。
- **パターンを毎回手で再現** → 半透明・モーションなど効果の数値が画面ごとに微妙に異なり、一貫性が崩れる。共通パターンで標準化する。
- **アクセシビリティを後回し** → 色コントラスト・モーション低減を後で対応しようとして抜け落ちる。トークン・パターンの段階で保証する。

> トークンのハードコード・原始トークンの直接参照のようなトークン次元の間違いは `design-system-tokens` の「よくある間違い」を見る。

## 4. チェックリスト
- [ ] トークンの値・命名・階層は `design-system-tokens` スキル（SoT）に従ったか（コンポーネントはセマンティックトークンのみ参照）
- [ ] ライト/ダークなどのテーマを**トークンの再定義**で切り替え、コンポーネントにテーマ分岐を撒き散らしていないか
- [ ] スタイルを**トークン/ユーティリティ → 共通変数 → ローカルスタイル**の優先順位で適用したか
- [ ] 繰り返すビジュアルパターン（半透明・モーションなど）を共通トークン/ユーティリティ/ミックスインで**再利用**しているか
- [ ] 色コントラスト・`prefers-reduced-motion` などの**アクセシビリティ**をトークン・パターンに組み込んだか

## 付録: スタック別の適用例

> 以下は参考用の実装例である。チームが使うスタック（例: React + Tailwind, CSS Modules, styled-components, Web Components など）に合う例を同じパターンで追加する。上記 1～4 の原則・規則が標準であり、付録はその適用事例にすぎない。

### Vue 3 + Vuetify

Vuetify のテーマシステムでグローバルテーマを一元管理し、ユーティリティクラスを第一優先のスタイル手段として使う。（このスタック固有の適用のみを扱う — テーマ切り替え・スタイルの優先順位の原則は §2-1, §2-2。）

#### テーマ設定 (`themeConfig.js`)
- ダーク/ライトモード、Primary カラーなどグローバルテーマ設定は `themeConfig.js` で一元管理する。（§2-1 の「一箇所でトークンを再定義」を Vuetify のテーマオブジェクトで実装した形。）

#### スタイルの優先順位 (§2-2 の Vuetify マッピング)
1. **Vuetify ユーティリティクラス**: Spacing（`pa-4`, `mt-2`）、Typography（`text-h5`）、Alignment などを優先的に使う。
2. **Sass 変数**: 共通に定義された SCSS 変数を参照する。
3. **Custom CSS**: やむを得ない場合にのみ `<style scoped>` を使う。

#### ダークモード対応 (Dark Mode)
- CSS 変数を使い、テーマ切り替え時に色と不透明度が自動で調整されるよう設計する。
- `[data-theme='dark']` セレクタを活用してグローバル変数を再定義する。（§2-1。）
