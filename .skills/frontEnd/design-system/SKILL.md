# 🎨 디자인 시스템 (Design System)

> 디자인 토큰을 **웹에 어떻게 적용하는가**를 다룬다 — 스타일 우선순위, 테마(라이트/다크) 전환, 공통 시각 패턴의 표준화. 새 화면/컴포넌트의 스타일을 정하거나 테마·다크 모드를 설계할 때, 하드코딩된 값을 토큰으로 정리할 때 읽는다. 특정 UI 프레임워크/라이브러리에 종속되지 않는 범용 표준이다.
>
> **토큰의 값·네이밍·계층(primitive→semantic→component)을 정의하는 권위(SoT)는 `design-system-tokens` 스킬이다.** 이 스킬은 그 토큰을 웹에서 소비하는 규칙만 다룬다.

## 1. 핵심 원칙
- **토큰 정의는 `design-system-tokens`를 따른다**: 토큰 단일 출처, 시맨틱 변수, primitive→semantic→component 계층, 네이밍은 그 스킬(SoT)의 규칙이다. 본 스킬은 정의된 토큰을 화면에서 참조하는 것만 다룬다 — 컴포넌트는 시맨틱 토큰만 칠하고 원시 값·원시 팔레트를 직접 박지 않는다.
- **테마는 토큰 재정의로 전환**: 라이트/다크 등 테마 차이는 시맨틱 토큰의 값을 테마별로 바꿔서 표현한다. 컴포넌트 코드는 테마를 몰라야 하고, 분기(`if dark`)를 흩뿌리지 않는다.
- **일관된 스타일 우선순위**: 토큰/시스템 유틸리티 → 공통 변수 → 컴포넌트 지역 스타일 순으로 적용한다. 같은 결정을 여러 계층에서 제각각 내리지 않는다.
- **공통 시각 패턴은 표준화**: 반복되는 시각 효과(반투명 표면, 모션/애니메이션 등)는 매번 새로 만들지 말고 공통 토큰·유틸리티·믹스인으로 정의해 재사용한다.
- **접근성은 기본값**: 색 대비·모션 축소(`prefers-reduced-motion`) 등 접근성 제약을 토큰·패턴 단계에서 보장한다.

> 임의 값(매직 넘버) 금지는 토큰 차원에서는 `design-system-tokens`(토큰에 없으면 토큰을 늘린다), 명명 상수 차원에서는 `coding-styles` 스킬을 따른다. 본 스킬의 §2-3은 이를 스타일 우선순위 관점에서만 다룬다.

## 2. 규칙

### 2-1. 테마(라이트/다크)는 토큰 재정의로 전환
- 테마 차이는 시맨틱 토큰 값의 차이로 표현하고, 테마 스위치는 한 곳(루트/테마 스코프)에서 토큰을 재정의해 건다.
- 컴포넌트 안에 `if (dark) ... else ...` 분기를 흩뿌리지 않는다. 색은 항상 같은 시맨틱 토큰을 칠한다. (토큰 이름·값 자체는 `design-system-tokens`를 따른다.)

```text
// ✅ 권장 — 테마별로 시맨틱 토큰 값만 재정의, 컴포넌트는 그대로
:root             { --surface-bg: #fff; --text-default: #111; }
[data-theme=dark] { --surface-bg: #111; --text-default: #eee; }

card { background: var(--surface-bg); color: var(--text-default); }  // 테마를 모른다

// ❌ 금지 — 컴포넌트가 테마를 알고 분기
card { background: isDark ? #111 : #fff; }
```

### 2-2. 스타일 우선순위를 지킨다
- 같은 스타일 결정을 여러 계층에서 중복으로 내리지 않는다. 우선순위:
  1. **디자인 시스템 토큰 / 표준 유틸리티** (간격·타이포·정렬 등 — 시스템이 제공하면 우선 사용)
  2. **공통 변수** (시맨틱 토큰/공통 스타일 변수 참조)
  3. **컴포넌트 지역 스타일** (위로 표현 못 하는 경우에만, 스코프를 좁혀 작성)
- 지역 스타일에서 토큰을 무시하고 값을 다시 박지 않는다.

```text
// ✅ 권장 — 시스템 유틸/토큰 우선, 부족분만 지역 스타일
<div class="stack gap-3">…</div>            // 표준 간격 유틸
.special { box-shadow: var(--shadow-md); }   // 토큰 참조한 지역 스타일

// ❌ 금지 — 지역 스타일에서 시스템을 우회해 매직 넘버
.special { margin-top: 13px; box-shadow: 0 2px 7px #0003; }
```

### 2-3. 공통 시각 패턴은 표준화해 재사용
- 반복되는 시각 효과(반투명/블러 표면, 모션 등)는 토큰·유틸리티·믹스인으로 정의해 재사용한다.
- 같은 효과를 화면마다 손으로 다시 만들지 않는다 — 값이 미세하게 어긋나 일관성이 깨진다.

```text
// ✅ 권장 — 패턴을 토큰/유틸로 표준화
.surface-glass {           // 반투명 표면 패턴을 한 번 정의해 재사용
  background: var(--surface-overlay);
  backdrop-filter: blur(var(--blur-md));
  border: 1px solid var(--border-subtle);
}

// ❌ 금지 — 화면마다 직접 수치를 박아 제각각
.panel { background: rgba(255,255,255,.15); backdrop-filter: blur(11px); }
```

### 2-4. 접근성을 패턴에 내장
- 텍스트/배경 색 대비가 기준(예: WCAG AA)을 만족하도록 시맨틱 토큰 쌍을 정의한다.
- 모션 패턴은 `prefers-reduced-motion`을 존중해, 사용자가 모션 축소를 원하면 비활성화한다.

```text
// ✅ 권장 — 모션 축소 선호를 존중
@media (prefers-reduced-motion: reduce) {
  * { animation: none; transition: none; }
}
```

## 3. 흔한 실수
- **컴포넌트가 테마를 분기** → 화면마다 `if dark`가 흩어져, 새 테마를 추가할 때마다 모든 컴포넌트를 고쳐야 한다. 토큰 재정의로 옮긴다.
- **우선순위 무시** → 시스템 유틸/토큰이 있는데 지역 스타일에서 값을 다시 박아 충돌·중복이 생긴다.
- **패턴을 매번 손으로 재현** → 반투명·모션 등 효과의 수치가 화면마다 미세하게 달라 일관성이 깨진다. 공통 패턴으로 표준화한다.
- **접근성 후순위** → 색 대비·모션 축소를 나중에 챙기다 빠뜨린다. 토큰·패턴 단계에서 보장한다.

> 토큰 하드코딩·원시 토큰 직접 참조 같은 토큰 차원의 실수는 `design-system-tokens`의 "흔한 실수"를 본다.

## 4. 체크리스트
- [ ] 토큰의 값·네이밍·계층은 `design-system-tokens` 스킬(SoT)을 따랐는가 (컴포넌트는 시맨틱 토큰만 참조)
- [ ] 라이트/다크 등 테마를 **토큰 재정의**로 전환하고, 컴포넌트에 테마 분기를 흩뿌리지 않는가
- [ ] 스타일을 **토큰/유틸 → 공통 변수 → 지역 스타일** 우선순위로 적용했는가
- [ ] 반복되는 시각 패턴(반투명·모션 등)을 공통 토큰/유틸/믹스인으로 **재사용**하는가
- [ ] 색 대비·`prefers-reduced-motion` 등 **접근성**을 토큰·패턴에 내장했는가

## 부록: 스택별 적용 예시

> 아래는 참고용 구현 예시다. 팀이 쓰는 스택(예: React + Tailwind, CSS Modules, styled-components, Web Components 등)에 맞는 예시를 같은 패턴으로 추가한다. 위 1~4의 원칙·규칙이 표준이고, 부록은 그 적용 사례일 뿐이다.

### Vue 3 + Vuetify

Vuetify 테마 시스템으로 전역 테마를 통합 관리하고, 유틸리티 클래스를 1순위 스타일 수단으로 쓴다. (이 스택 고유의 적용만 다룬다 — 테마 전환·스타일 우선순위의 원칙은 §2-1, §2-2.)

#### 테마 설정 (`themeConfig.js`)
- 다크/라이트 모드, Primary 컬러 등 전역 테마 설정은 `themeConfig.js`에서 통합 관리한다. (§2-1의 "한 곳에서 토큰 재정의"를 Vuetify 테마 객체로 구현한 형태.)

#### 스타일 우선순위 (§2-2의 Vuetify 매핑)
1. **Vuetify 유틸리티 클래스**: Spacing(`pa-4`, `mt-2`), Typography(`text-h5`), Alignment 등을 우선적으로 사용한다.
2. **Sass 변수**: 공통으로 정의된 SCSS 변수를 참조한다.
3. **Custom CSS**: 불가피한 경우에만 `<style scoped>`를 사용한다.

#### 다크 모드 대응 (Dark Mode)
- CSS 변수를 사용하여 테마 전환 시 색상과 투명도가 자동으로 조절되도록 설계한다.
- `[data-theme='dark']` 셀렉터를 활용하여 전역 변수를 재정의한다. (§2-1.)
